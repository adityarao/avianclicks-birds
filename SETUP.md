# eBird Grid – Setup Guide

Complete walkthrough: GitHub repo → nightly action → GoDaddy embed.

---

## 1. Create a GitHub repository

1. Go to github.com → **New repository**
2. Name it something like `avianclicks-birds` — make it **Public** (so `raw.githubusercontent.com` URLs work without auth)
3. Clone it locally or use GitHub's web editor

Repo structure you'll end up with:
```
avianclicks-birds/
├── .github/
│   └── workflows/
│       └── update-birds.yml   ← GitHub Action
├── scripts/
│   └── process_ebird.py       ← processing script
├── MyEBirdData.csv            ← your eBird export (you commit this)
└── birds.json                 ← auto-generated by the action
```

---

## 2. Add your eBird API key as a secret

1. In your GitHub repo → **Settings** → **Secrets and variables** → **Actions**
2. Click **New repository secret**
3. Name: `EBIRD_API_KEY`
4. Value: your eBird API key (get one free at https://ebird.org/api/keygen)
5. Click **Add secret**

---

## 3. Export your eBird India data

1. Log into ebird.org → **My eBird** → **Download My Data**
2. You'll get a file called `MyEBirdData.csv`
   - This contains ALL your observations worldwide
   - The script automatically filters to India (State/Province codes starting with `IN-`)
   - Alternatively, filter to India on eBird before downloading
3. Commit this file to the **root** of your GitHub repo as `MyEBirdData.csv`

> **When you tick new birds**: re-download from eBird, replace `MyEBirdData.csv` in the repo, and either wait for the nightly run or trigger the action manually.

---

## 4. Add the scripts and workflow

Copy these files from this folder into your repo, matching the same paths:

| This file | → Repo path |
|---|---|
| `scripts/process_ebird.py` | `scripts/process_ebird.py` |
| `.github/workflows/update-birds.yml` | `.github/workflows/update-birds.yml` |

---

## 5. Run the action for the first time

1. In your repo → **Actions** tab
2. Click **Update Birds JSON** in the left sidebar
3. Click **Run workflow** → **Run workflow** (green button)
4. Watch the run — it should take ~30 seconds
5. After it completes, `birds.json` will appear in your repo root

If it fails, check the logs — most common issue is the CSV not being found or the API key secret being named wrong.

---

## 6. Update the HTML file

Open `birds-grid.html` and find this line near the top of the `<script>` block:

```js
const BIRDS_JSON_URL =
  "https://raw.githubusercontent.com/YOUR_GITHUB_USERNAME/YOUR_REPO_NAME/main/birds.json";
```

Replace with your actual values, e.g.:
```js
const BIRDS_JSON_URL =
  "https://raw.githubusercontent.com/adityarrao/avianclicks-birds/main/birds.json";
```

---

## 7. Embed in GoDaddy Website Builder

1. In GoDaddy Website Builder, go to the page where you want the bird list
2. Add a **HTML** section / **Custom Code** block
3. Paste the **entire contents** of `birds-grid.html` into it
4. Save and preview

> **GoDaddy iframe note**: Some GoDaddy builders wrap custom HTML in an iframe — if the page looks cut off, look for a "height" setting on the custom code block and set it to something large (e.g. 2000px), or use `100vh`.

---

## Schedule

The action runs at **00:30 UTC (06:00 IST) every night**. Since your India list only changes when you go birding and submit checklists, nightly is more than frequent enough. You can also trigger it manually anytime from the Actions tab.

---

## Troubleshooting

| Symptom | Fix |
|---|---|
| Action fails: `CSV not found` | Make sure `MyEBirdData.csv` is in the repo root (not in a subfolder) |
| Action fails: `Error: eBird API key required` | Check the secret name is exactly `EBIRD_API_KEY` |
| Page shows "Could not load bird list" | Check the `BIRDS_JSON_URL` in the HTML matches your repo exactly |
| Species missing from list | They may have a subspecies scientific name in your CSV that doesn't match taxonomy — check the action log for "unmatched" warnings |
| GoDaddy page looks blank | The custom code block may need a fixed height — try setting it to `1800px` or more |
